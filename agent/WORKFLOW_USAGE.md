## 基本概念

### Workflow（工作流）

工作流是邮件发送的**模板定义**，描述了邮件发送的步骤和逻辑。

- 邮件节点：发送邮件
- 驱动节点：等待事件触发（打开邮件、点击链接）
- 延时节点：延迟一段时间后再发送

### WorkflowInstance（工作流实例）

工作流实例是工作流的**具体执行**。每次执行工作流时，会为每个收件人创建一个独立的实例。

**实例状态**:
- `pending`: 待执行
- `running`: 运行中
- `waiting_event`: 等待事件（如邮件打开、点击）
- `delayed`: 已延时（等待调度器执行）
- `completed`: 已完成
- `failed`: 失败
- `cancelled`: 已取消

## 创建工作流

### 步骤 1: 进入工作流页面

1. 登录 Mail Flow
2. 点击左侧菜单 "节点发送"

### 步骤 2: 配置工作流属性

1. 双击顶部输入框编辑工作流名称
2. 选择执行模式：
   - **手动**: 用户点击"发送"按钮执行
   - **定时**: 在指定时间自动执行

### 步骤 3: 添加节点

点击"添加节点"按钮，选择节点类型：

#### 邮件节点

- **节点名称**: 描述性名称
- **选择模板**: 选择邮件模板
- **收件人类型**:
  - 联系人：选择具体联系人
  - 用户组：选择整个用户组

#### 驱动节点（事件驱动）

驱动节点等待特定邮件事件后触发后续动作。

**步骤配置**:

1. **事件监听**（必选）
   - 事件类型：open（打开）、click（点击）、delivery（送达）、bounce（退信）
   - 链接URL：如果是 click 事件，可指定特定链接

2. **条件判断**（可选）
   - 字段：event_type（事件类型）、link_url（链接URL）、recipient（收件人）
   - 运算符：eq（等于）、neq（不等于）、contains（包含）、not_contains（不包含）
   - 值：比较值

3. **延时执行**（可选）
   - 相对延时：等待 X 分钟/小时/天后执行
   - 绝对延时：指定具体日期时间

### 步骤 4: 连接节点

在画布上拖拽节点，使用连线工具连接节点：
- 拖动起点（节点底部圆点）到终点（节点顶部圆点）
- 箭头的方向表示执行顺序

### 步骤 5: 保存工作流

点击"保存"按钮，工作流会出现在已保存列表中。

## 工作流节点

### 邮件节点

```
[邮件节点] ──▶ 发送邮件给指定收件人
```

**配置项**:
- 邮件模板
- 收件人（联系人或用户组）
- 节点名称

**执行行为**:
- 立即发送邮件
- 记录 EmailLog
- 继续执行后续节点

### 驱动节点

```
[驱动节点] ──▶ 等待事件 ──▶ 条件判断 ──▶ 延时 ──▶ 继续执行
```

**执行行为**:
1. 实例状态变为 `waiting_event`
2. 等待 AWS SES/SNS 事件
3. 收到事件后评估条件
4. 条件满足：按配置延时后继续
5. 条件不满足：实例结束

### 延时节点

```
[延时节点] ──▶ 延迟 X 时间 ──▶ 继续执行
```

**配置项**:
- 延时类型：相对/绝对
- 延时值

**执行行为**:
- 实例状态变为 `delayed`
- 创建调度任务
- 延时结束后恢复执行

## 执行工作流

### 手动执行

1. 在已保存列表中点击工作流名称加载
2. 点击"发送"按钮
3. 系统会为每个收件人创建独立的 WorkflowInstance
4. 查看执行结果

### 定时执行

1. 编辑工作流属性
2. 设置执行模式为"定时"
3. 选择执行时间
4. 保存工作流
5. 系统会在指定时间自动执行

## 管理工作流实例

### 查看实例列表

1. 点击左侧菜单"工作流实例"
2. 可以看到所有实例的状态概览

**筛选功能**:
- 按状态筛选（待执行、运行中、等待事件等）
- 搜索工作流名称
- 搜索收件人邮箱

### 查看实例详情

点击"查看"按钮，可以看到：

1. **基本信息**
   - 实例ID
   - 工作流名称
   - 收件人
   - 邮件Message ID
   - 当前状态
   - 当前节点

2. **等待条件**（如果状态为 waiting_event）
   - 等待的事件类型
   - 条件配置
   - 等待开始时间

3. **邮件记录**
   - 已发送的邮件列表
   - 发送状态

4. **事件记录**
   - 收到的邮件事件

### 取消实例

如果实例处于等待状态（waiting_event、delayed、pending），可以点击"取消"按钮提前终止。

## 常用场景示例

### 场景 1: 欢迎邮件

**需求**: 新用户注册后发送欢迎邮件

**工作流**:
```
[开始] ──▶ [邮件节点: 欢迎邮件] ──▶ [结束]
```

**配置**:
- 邮件节点选择"欢迎邮件"模板
- 选择目标联系人

---

### 场景 2: 营销邮件（打开追踪）

**需求**: 发送营销邮件，如果用户24小时内打开，则发送后续优惠邮件

**工作流**:
```
[开始] ──▶ [邮件: 营销邮件] ──▶ [驱动节点: 等待打开事件] ──▶ [邮件: 优惠邮件] ──▶ [结束]
```

**驱动节点配置**:
- 事件类型：open
- 延时：24小时后

---

### 场景 3: 购物车提醒（点击追踪）

**需求**: 发送购物车提醒邮件，如果用户点击"继续购物"链接，发送优惠券

**工作流**:
```
[开始] ──▶ [邮件: 购物车提醒] ──▶ [驱动节点: 等待点击特定链接] ──▶ [邮件: 优惠券] ──▶ [结束]
```

**驱动节点配置**:
- 事件类型：click
- 链接URL：包含"continue-shopping"
- 条件：link_url contains "continue-shopping"

---

### 场景 4: 生日祝福

**需求**: 在用户生日当天自动发送祝福邮件

**工作流**:
```
[开始] ──▶ [邮件: 生日祝福] ──▶ [结束]
```

**配置**:
- 执行模式：定时
- 执行时间：用户生日日期

---

### 场景 5: 多阶段培育

**需求**: 7天邮件培育序列

**工作流**:
```
[邮件: 介绍]
    │
    ▼
[延时: 2天]
    │
    ▼
[邮件: 详细功能介绍]
    │
    ▼
[延时: 3天]
    │
    ▼
[邮件: 案例分享]
    │
    ▼
[延时: 2天]
    │
    ▼
[邮件: 行动号召]
```

## API 参考

### 执行工作流

```http
POST /api/workflow/{workflow_id}/execute
Content-Type: application/json
Authorization: Bearer {token}

{
  "mock": false  // 设置为 true 模拟发送，不实际发送邮件
}
```

**响应**:
```json
{
  "msg": "工作流执行已启动",
  "instance_count": 3,
  "instances": [
    {"id": 1, "recipient_email": "user1@example.com", "status": "running"},
    {"id": 2, "recipient_email": "user2@example.com", "status": "running"},
    {"id": 3, "recipient_email": "user3@example.com", "status": "running"}
  ]
}
```

### 获取实例列表

```http
GET /api/user/instances?status=waiting_event&page=1&per_page=20
Authorization: Bearer {token}
```

### 获取实例详情

```http
GET /api/instance/{instance_id}
Authorization: Bearer {token}
```

### 取消实例

```http
POST /api/instance/{instance_id}/cancel
Authorization: Bearer {token}
```

## 常见问题

### Q: 工作流执行后实例状态一直是 pending？

A: 检查后端日志，可能是：
- APScheduler 未启动
- 数据库连接问题
- 代码异常

### Q: 驱动节点没有收到事件？

A: 检查：
- AWS SNS 配置是否正确（参考 SNS_SETUP.md）
- 邮件是否已送达
- Webhook URL 是否可访问

### Q: 延时节点没有按时执行？

A: 检查：
- APScheduler 是否使用 SQLAlchemyJobStore
- 数据库是否正常
- 调度器是否运行

### Q: 如何调试工作流？

A: 使用 mock 模式：
```json
{
  "mock": true
}
```
这会创建实例但不实际发送邮件。
